
### Glosario de Términos Clave de Formularios

| Término | Definición |
|---------|------------|
| `<fieldset>` | Etiqueta HTML que se utiliza para agrupar controles y etiquetas relacionados dentro de un formulario, mostrando típicamente un recuadro a su alrededor. |
| `<legend>` | Etiqueta que define un título o leyenda para un elemento `<fieldset>`, el cual se muestra en el borde del recuadro. |
| `input type="number"` | Un control de formulario para introducir valores numéricos. |
| `input type="email"` | Un control de formulario diseñado específicamente para la entrada de direcciones de correo electrónico, validando su formato. |
| `input type="url"` | Un control de formulario para la entrada de direcciones web (URL). |
| `input type="date"` | Un control de formulario que permite al usuario seleccionar una fecha de un calendario. |
| `input type="time"` | Un control de formulario para introducir una hora. |
| `input type="datetime-local"` | Un control de formulario para seleccionar una fecha y hora. |
| `input type="month"` | Un control de formulario para seleccionar un mes y un año. |
| `input type="search"` | Un campo de texto de una sola línea diseñado para la introducción de términos de búsqueda. |
| `input type="color"` | Un control que presenta un selector de colores para que el usuario elija un color. |
| `action` | Atributo de la etiqueta `<form>` que contiene la URL de la página o servicio (script) que procesará los datos del formulario. |
| `checked` | Atributo booleano utilizado en `input type="radio"` y `input type="checkbox"` para indicar que un elemento debe aparecer seleccionado por defecto. |
| `checkbox` | Tipo de `input` que crea una casilla de verificación. Se utiliza para representar valores booleanos (verdadero/falso), donde marcarla equivale a "verdadero". |
| `cols` | Atributo de la etiqueta `<textarea>` que especifica el número de columnas (ancho en caracteres) del área de texto. |
| `file` | Tipo de `input` que permite al usuario seleccionar un fichero de su dispositivo de almacenamiento (disco duro, pendrive, etc.) para subirlo. |
| `<form>` | Etiqueta básica de HTML que engloba todos los elementos de un formulario, definiendo el inicio y el fin de la sección interactiva. |
| `get` | Uno de los dos valores para el atributo `method`. Transmite los datos del formulario de forma visible en la URL de la página de destino. |
| `id` | Atributo que sirve como identificador único para un elemento en el documento. W3C aboga por su uso sobre `name`. |
| `<input>` | Etiqueta fundamental para crear la mayoría de los campos de entrada de un formulario, como cajas de texto, botones y casillas de verificación. |
| `maxlength` | Atributo para `input type="text"` que define el número máximo de caracteres que el usuario puede escribir en el campo. |
| `method` | Atributo de la etiqueta `<form>` que especifica la forma en que se transmitirán los datos al servidor. Sus valores comunes son `get` y `post`. |
| `name` | Atributo utilizado para identificar los datos del formulario cuando son procesados por un script del lado del servidor, como PHP. |
| `<optgroup>` | Etiqueta utilizada dentro de un `<select>` para agrupar opciones relacionadas bajo una etiqueta común (por ejemplo, agrupar equipos por división). |
| `password` | Tipo de `input` que crea un campo de texto donde los caracteres introducidos por el usuario se enmascaran (muestran como puntos o asteriscos). |
| `PHP` | Lenguaje de programación del lado del servidor mencionado como el encargado habitual de procesar la información que un usuario envía a través de un formulario. |
| `post` | Uno de los dos valores para el atributo `method`. Transmite los datos del formulario de forma no visible en la URL, siendo el método preferido para datos sensibles. |
| `radio` | Tipo de `input` que crea un "botón de opción". Permite al usuario seleccionar una única opción de un grupo de opciones mutuamente excluyentes. |
| `reset` | Tipo de `input` que crea un botón para reiniciar todos los campos del formulario a sus valores iniciales. |
| `rows` | Atributo de la etiqueta `<textarea>` que especifica el número de filas (altura) del área de texto. |
| `<select>` | Etiqueta que crea una lista desplegable o cuadro combinado, permitiendo al usuario escoger una o varias opciones de una lista predefinida. |
| `size` | Atributo para `input type="text"` que controla el tamaño visible (ancho en caracteres) del cuadro de texto en la página. |
| `submit` | Tipo de `input` que crea un botón para enviar los datos del formulario al servidor especificado en el atributo `action`. |
| `<textarea>` | Etiqueta que crea un cuadro de texto multilínea, útil para que el usuario pueda introducir textos largos. |
| `text` | Tipo de `input` básico que crea una caja de texto de una sola línea para la introducción de información. |
| `value` | Atributo polivalente. En los `input` de texto, define el valor inicial; en los botones (`submit`, `reset`), define el texto que se muestra en ellos. |
| `<label>` | Etiqueta que representa una descripción o título para un control en un formulario. |
| `for (atributo)` | Atributo de la etiqueta `<label>` que se utiliza para asociarla explícitamente a un control de formulario mediante el `id` de dicho control. |
| Patrones regulares | Secuencias de caracteres que forman un patrón de búsqueda. En el contexto de los formularios, se utilizan para validar que los datos introducidos por el usuario (como un email) sigan un formato específico. |
| `<datalist>` | Etiqueta que contiene un conjunto de opciones predefinidas para un control de entrada, proporcionando una funcionalidad de autocompletado. |
